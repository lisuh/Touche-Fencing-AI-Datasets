# Touché

Last researched: **June 20, 2026**. Items are listed in order from *strongest/most likely to contribute to research* to the *weakest/least likely to contribute to research*.

A list of public datasets and dataset-like resources for AI, machine learning, computer vision, pose estimation, action recognition, match analytics, rankings, and referee-assistance work for the Olympic sport of fencing. Resources only need to be publicly visible to be listed, but weakly documented or non-downloadable entries are labeled clearly. If you find a new dataset, a working download link, or updated release status for an unverified item, please update the repository.

Status tags:

- **Dataset**: public data files or a working dataset page were found.
- **General dataset**: a broader dataset containing at least one fencing-related class.
- **Benchmark / paper release**: a paper describes a benchmark or release, but a standalone download may need verification.
- **Code / pipeline**: public collection, scraping, processing, modeling, or analytics code rather than a formal dataset.
- **Asset / test data**: useful public artifacts, fixtures, or model files, but not a complete training dataset.
- **Low documentation**: visible files exist, but provenance, schema, or license needs investigation.

## Fencing-specific video, image, pose, and sensor datasets

1. [Fencing Footwork Dataset (FFD)](https://home.agh.edu.pl/~fmal/ffd/): **Dataset.** Six footwork actions from 10 fencers, recorded with Kinect and x-IMU. It includes skeletons, depth video, accelerometer, gyroscope, magnetometer, and rotation data, but no RGB. The official page provides a Google Drive download of about 1 GB compressed and more than 30 GB extracted. Strongest public multimodal footwork resource found. [[FenceNet paper]](https://openaccess.thecvf.com/content/CVPR2022W/CVSports/papers/Zhu_FenceNet_Fine-Grained_Footwork_Recognition_in_Fencing_CVPRW_2022_paper.pdf)
2. [Sport : Fencing Matches AI](https://www.kaggle.com/datasets/alexpgd/sport-fencing-matches-ai): **Dataset.** About 855 MB of fencing-assault data intended for outcome prediction from fencer movement. Kaggle lists the license as unknown, so it is useful for experimentation but risky for redistribution or production training without clarification.
3. [Few-shot FIE epee movement dataset](https://github.com/d-mali/fie-epee-moves-clasificator): **Dataset.** Contains 130 annotated clips from world-level epee competitions across seven movement classes: fleche, jump, direct lunge, jumping lunge, squat, step backward, and step forward. The repo includes raw MP4 clips, processed clips, NumPy point-tracking data, and JSON annotations. Very useful for movement classification, but no repository license was found.
4. [CMU Fencing Touch Detection Images](https://huggingface.co/datasets/mastefan/2025-24679-Project-One-Fencing-Score-Detection): **Dataset.** Image classification data with `no_touch` and `touch` labels: 1,116 original examples and 5,580 augmented examples. Useful for detecting scoring-light events and automatically segmenting long bout videos. No explicit license was visible in the dataset card metadata.
5. [CMU Fencing Score Detection Time Series](https://huggingface.co/datasets/emkessle/2025-24679-Project-One-Fencing-Score-Detection-Timeseries-labeled): **Dataset.** Roughly 2,200 labeled frames with video/frame IDs, timestamps, red and green ratios, and a decaying score-light label. Derived from public Quarte Riposte footage and described as MIT-licensed. Useful for touch-event timing and scoreboard-light signal modeling.
6. [CMU Fencing Scoreboard YOLOv8 Dataset](https://huggingface.co/datasets/mastefan/fencing-scoreboard-yolov8): **Dataset.** Around 2,000 scoreboard-detection images, split into about 1,600 training and 400 validation samples, with YOLO bounding boxes. The card describes 50 lit and 50 unlit originals, augmentation, and 1,000 negative frames. Useful for locating scoring boxes before reading lights or scores; the card says MIT.
7. [Log45 Fencing-Judge-AI clips](https://github.com/Log45/Fencing-Judge-AI): **Dataset / pipeline.** The README links a public Google Drive folder containing clips from World Cups, Olympics, and college club fencing and reports about 200 touch-length clips during development. Useful for saber phrase classification, but clip-level labels, source-video rights, and a dataset license need verification. [[Google Drive data]](https://drive.google.com/drive/folders/1VmFhBmm88Z9LU0zVJsLevOETzgLU8qim?usp=sharing)
8. [FencingVision pre-dataset](https://github.com/Ph1n-Pham/fencing-vision): **Dataset / code.** Public repository containing fencing videos, a COCO annotation JSON, an instance-segmentation model, and inference examples for fencer body-part segmentation. Useful as a compact segmentation starter set; documentation does not clearly state the number of labeled frames or the footage license.
9. [FENCINGLY](https://huggingface.co/datasets/EXCALOFRIO/FENCINGLY): **Low documentation.** Apache-2.0 Hugging Face repository containing two compressed JSON archives plus custom dataset and transformation code. It appears intended for pose/action modeling, but the dataset card has no description, schema, counts, or provenance.
10. [LeRobot fencing imitation dataset](https://huggingface.co/datasets/ashishrao7/fencing-lerobot-realsense-depth-and-selfie-cam): **Dataset.** Three robot-imitation episodes with synchronized multi-camera videos and Parquet state/action data. Its task is to react to a fencer with block/block/stab behavior. This is robotics data rather than competitive-fencing bout data, but it is publicly downloadable and relevant to embodied fencing interaction.

## Competition, bout, ranking, and athlete datasets

1. [USA Fencing National Tournament Dataset](https://www.kaggle.com/datasets/rivkalipkovitz/usa-fencing-national-tournament-dataset): **Dataset.** Results from USA Fencing national events from 2016-2020, licensed AGPL-3.0. It contains 34 tournaments, 1,089 events, 98,274 event entries, 14,714 pools, 290,172 pool bouts, 84,371 direct-elimination bouts, and 1,319 referees. The CSV schema links tournaments, events, entries, bouts, pools, and referee assignments. This is the strongest public US competition-results dataset found.
2. [FIE Fencing Dataset](https://github.com/amichaelsen/fie-fencing-dataset): **Dataset.** GPL-3.0 scraper plus committed CSV exports for men's and women's foil. Files cover tournaments, pool bouts, fencer biographies, and historical rankings. The women's Kaggle release describes nearly 50,000 bouts from 218 tournaments and 2,108 fencers from January 2014 to May 2021. [[CC0 Kaggle mirror for women's foil]](https://www.kaggle.com/datasets/amichaelsen/fie-fencing-womens-foil-data)
3. [Sabre Fencing Win and Scoreline Prediction data](https://github.com/KahHongg/Fencing_win_and_score_predictor): **Dataset / code.** Contains public CSVs for six men's sabre events from the 2023/2024 cycle, including athletes, pools, elimination rounds, and final rankings. Useful for recent FIE head-to-head and score modeling. No repository license was found.
4. [Fencing-Points-MF](https://www.kaggle.com/datasets/foillee/fencing-points): **Dataset.** MIT-licensed time series of points earned by most international senior men's foil fencers. Designed for career-trajectory visualization and future-performance forecasting. Small but directly useful for ranking trend charts.
5. [USA Fencing Team Archives](https://www.kaggle.com/datasets/ajaxofgreee/usa-fencing-team-archives): **Dataset.** Historical snapshots of USA Fencing team rankings, reportedly going back to 2010. Useful for ranking movement and team-selection history, but the dataset has minimal documentation and an unknown license.
6. [2024 NCAA Fencing National Championship](https://www.kaggle.com/datasets/mexwell/2024-ncaa-fencing-national-championship): **Dataset.** A 144-row table covering men and women in all three weapons. Fields include placement, school, victories, bouts, win percentage, touches scored/received, indicator, gender, and Ivy status. Useful for collegiate analysis, but it contains tournament totals rather than opponent-level bout records.
7. [US Fencing Population Study](https://github.com/wesleyyuan17/us-fencing-population-study): **Dataset / analysis.** Includes roughly 53 MB of raw AskFRED-derived data and 56 MB of processed data for studying US fencing participation. The author warns that coverage is incomplete, restricted toward senior/open events, and distorted after 2020. Useful for population trends with careful caveats; no license was found.
8. [Fencing at the Summer Olympics](https://www.kaggle.com/datasets/mathurinache/fencing-at-the-summer-olympics): **Dataset.** Tiny CC0 CSV extracted from the corresponding Wikipedia page. Best treated as a simple historical medal/event lookup rather than a bout dataset.
9. [Rio 2016 athlete and event data](https://github.com/flother/rio2016): **General dataset.** CSVs for more than 11,500 athletes and 306 Rio events, including fencing participants and events. Useful for Olympic participation joins, but not for bout-level fencing analysis.
10. [Olympic Summer and Winter Games, 1896-2022](https://www.kaggle.com/datasets/piterfm/olympic-games-medals-19862018): **General dataset.** Athlete, result, and medal history across Olympic sports with a fencing subset. Licensed CC BY-NC-SA 4.0. Useful for long-term country, athlete, and medal context.
11. [Olympic-fencing](https://github.com/Zargon42/Olympic-fencing): **Dataset / analysis.** Regression project that commits cleaned women's foil bout, biography, tournament, and country CSVs derived from the Michaelsen FIE data. It is a convenient analysis-ready subset, not an independent source; no license was found.
12. [samasas/fencing](https://www.kaggle.com/datasets/samasas/fencing): **Low documentation.** Approximately 18 KB and marked Apache-2.0, but has no public description or schema. Inspect files before listing it as substantive data.
13. [rongdianzhu/fencing](https://www.kaggle.com/datasets/rongdianzhu/fencing): **Low documentation.** Approximately 2 KB with no description and an unknown license. Publicly visible, but not currently useful without manual inspection.

## Benchmarks and research releases

1. [FACTS - Fine-Grained Action Classification for Tactical Sports](https://arxiv.org/abs/2412.16454): **Benchmark / paper release.** Introduces a fencing and boxing benchmark. The fencing portion is described as 6,400 final clips from 13,459 annotated Quarte Riposte clips, using eight labels: attack, riposte, counter-attack, and remise for left and right. The paper says data are public, but no standalone download URL was found in the arXiv metadata or indexed results.
2. [FERA - Foil Fencing Referee Assistant](https://arxiv.org/abs/2509.18527): **Benchmark / paper release.** Reports 493 underlying clip IDs, 598 pose-view files, 1,800 cleaned segments, 11 move labels, and five blade-position labels. It says anonymized pose features, labels, and code are released, but no direct public repository or download was found.
3. [VirtualFencer](https://arxiv.org/abs/2507.00261): **Research lead.** Extracts 3D motion and strategy from in-the-wild fencing videos and generates bouts. No public dataset release was found, but the task and representation are relevant to motion and tactical modeling.
4. [FenceNet](https://arxiv.org/abs/2204.09434): **Related paper.** Not a separate dataset; it uses FFD and reports 652 videos, 10 fencers, six footwork classes, and multimodal skeleton/depth/IMU inputs.

## Code, collection pipelines, and public data tools

1. [RT-Fencing](https://github.com/CodLiver/RT-Fencing): **Code / pipeline.** Verified public MIT repository for touch-time fencing move classification. Its README links model states and related files on Google Drive and says source videos came from the Creative Commons non-commercial Fencing Vision YouTube channel. This is a real dataset lead, but the release is not packaged as a conventional benchmark. [[Google Drive]](https://drive.google.com/drive/folders/1IHlRLp2y5sltFbCKqQoKaFS5xLk9gGB4?usp=sharing)
2. [sholtodouglas/fencing-AI](https://github.com/sholtodouglas/fencing-AI): **Code / pipeline.** Downloads match videos, cuts clips before scoring-light events, derives labels from scoreboard changes, performs augmentation, and trains referee models. The README reports roughly 5,500 clips or 11,000 after flipping, but the generated dataset is not released.
3. [GalDude33/fencing-AI](https://github.com/GalDude33/fencing-AI): **Code / pipeline.** Pose, optical-flow, and C3D-style extension of the fencing-referee workflow. Builds 50-frame clips before scoring-light events and labels them from score differences; no formal dataset release.
4. [fencing-insights / fencestats](https://github.com/s-eun-young-g/fencing-insights): **Code / analytics pipeline.** Parses public FencingTracker bout histories and computes head-to-head records, one-touch results, seed over/under-performance, DE upsets, location splits, streaks, touch differential, and expected wins. It deliberately ships synthetic/anonymized examples rather than third-party data.
5. [FREAKS / Fencing_data](https://github.com/caroolus/Fencing_data): **Code / live data pipeline.** Django app that aggregates FIE rankings and Fencing Time Live links across weapons, genders, categories, and seasons. Useful architecture and ingestion reference; the repository does not commit a substantive dataset.
6. [AskFRED v1 Archive](https://github.com/boscojwho/askFRED-v1-archive): **Code / archive.** Large public archive of the retired AskFRED v1 site and REST API documentation, including filter/sort schemas. Useful for understanding historical IDs and ingestion behavior, but it is not a clean results export.
7. [FencingResultsAggregate](https://github.com/Fencerman2/FencingResultsAggregate): **Code / pipeline.** AskFRED-based utility for head-to-head, pool/DE, point, and indicator analysis. No bundled dataset.
8. [ThePiste](https://github.com/ianswope/ThePiste.org): **Code / pipeline.** Public tournament-planning app with AskFRED synchronization, CSV import, eligibility logic, travel scoring, goals, and results tracking. Useful for tournament-catalog schema ideas, not as a dataset source.
9. [Saber Referee Assistant](https://github.com/LeJamon/saber-referee): **Code / asset data.** End-to-end pose/blade/event/right-of-way pipeline with 13 small JSON phrase fixtures covering calls such as parry-riposte, simultaneous, point-in-line, and low-confidence abstention. Fixtures are synthetic tests, but they are useful for validating rule engines.
10. [FencingMaestro](https://github.com/cinastanbean/FencingMaestro): **Code / pipeline.** Public video-analysis GUI and motion-analysis scaffold. Action-recognition training remains roadmap work and no dataset is bundled.

## General AI datasets containing fencing-related classes

1. [UCF101](https://www.crcv.ucf.edu/data/UCF101.php): **General dataset.** Public action-recognition benchmark with 13,320 YouTube videos across 101 classes, including a dedicated `Fencing` class. Useful for pretraining or baseline fencing-action recognition, although the clips are coarse action labels rather than referee-decision labels.
2. [HMDB51](https://serre-lab.clps.brown.edu/resource/hmdb-a-large-human-motion-database/): **General dataset.** Human-motion benchmark containing `fencing`, `draw_sword`, `sword`, and `sword_exercise` classes. Useful for transfer learning and robustness to movie-like footage; only the `fencing` class is directly sport-specific.
3. [Sports Image - Image Classification](https://www.kaggle.com/datasets/sheikhzaib/sports-image-image-classification): **General dataset.** CC0 image dataset with 22 sport folders, including fencing. Useful for sport-scene classification or sourcing generic fencing images, but not for temporal action understanding.
4. [Kinetics / I3D labels](https://github.com/google-deepmind/kinetics-i3d): **General dataset.** Kinetics-400 includes a `sword fighting` class rather than a dedicated sport-fencing label. It is only loosely related, but may help transfer learning for two-person weapon motion.

## Notes for contributors

- Prefer direct data URLs with a clear license, schema, provenance, and citation.
- Separate source datasets from mirrors, derived subsets, trained models, and code-only repositories.
- Record whether footage may be redistributed and whether its license permits commercial use.
- Flag identity fields involving minors, birthdays, clubs, or membership data for privacy review.
- Mark claimed releases as **Benchmark / paper release** until a working public download is verified.
